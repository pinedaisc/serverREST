Iniciando una aplicación Java en Heroku

Este es un ejemplo de como desplegar una aplicación Java en Heroku. Es una aplicación razonablemente simple - pero una buena base para entender lo más basico de la plataforma Heroku.

Comenzando con Java fuente en GitHub
para desplegar tu propia copia, y aprender los fundamentos de la plataforma Heroku, revisa el tutorialIniciando con Java en Heroku.

How this sample app works

  • Esta aplicación fue desplegada en Heroku, ya se sea usando Git o Botón Heroku en el repositorio.
  • Cuando Heroku recibe el codigo fuente, se trae todas las refencias del archivopom.xml, Creando un gusano.
  • La plataforma gira alrededor de los dyno, un contenedor ligero que provee un ambiente aislado en el que el gunasano puede ser desplegado y ejecutado.
  • Puedes escalar tu aplicacion, manejarla y deployarla 150 servicios para agregar, desde la consola de administracion o el CLI.

Next Steps

  • si estás siguiendo la guia Comenzando, entonces por favor revisa bien el tutorial y realiza los próximos pasos!
  • Si desplegaste esta aplicación desde el boton en Heroku, en una linea de comandos, ejecuta:
    • git clone https://github.com/heroku/java-getting-started.git - esto creará una copia del codigo fuente de tu aplicación
    • cd java-getting-started - cambia al directorio dentro de tu codigo fuente en tu repositorio
    • heroku git:remote -a <your-app-name> - asosciar la aplicacion remota con tu copia en local
    • ahora deberias configurar la aplicación para correr localmente desplegar cambios a Heroku
